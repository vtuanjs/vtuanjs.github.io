# Project memory — writing preferences

Durable preferences for working on this blog, committed to the repo so they apply on any machine. This is an index only — full rules, checklists, and examples live in `.claude/skills/write-blog*`. Do not duplicate skill content here.

- **Layered depth** — every post serves a scanner and a reader who stays; never trim depth to gain scannability. See `write-blog*` → "Layered depth."
- **Visual-first** — structure is carried by animated diagram components in `src/components/diagrams/`, not prose walls; Mermaid is the fallback. See `write-blog-technical` → "Visual-first"; reference post `src/content/technical/the-dual-write-problem.mdx`.
- **Tight prose** — every sentence earns its place; cut anything that can go without losing an idea. See `write-blog*` → "Tight prose."
- **No paper walls** — inside every section, parallel items/steps/rules/before-after are lists or tables, not paragraphs. See `write-blog*` → "Format inside sections — no paper walls."
- **No emoji, natural voice** — no emoji anywhere; avoid AI tells. See `write-blog*` → voice rules.
- **Technical register** — Technical-space posts use precise terms of art, no decorative/emotional flourish. See `write-blog-technical` → "Technical language rules."
